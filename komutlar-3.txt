# 🔄 CI/CD PIPELINE: GitHub Actions

### 📄 `.github/workflows/ci-cd.yml`
Monorepo-aware, paralel çalıştırma destekli, güvenlik taramalı, GHCR (GitHub Container Registry) entegrasyonlu ve environment-gate'li production pipeline.

```yaml
# .github/workflows/ci-cd.yml
name: CI/CD Pipeline
on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

env:
  NODE_VERSION: '20'
  REGISTRY: ghcr.io/${{ github.repository_owner }}
  HELM_CHART_PATH: packages/infra/helm/planner-ecosystem

jobs:
  # ==================== 1. VALIDATION ====================
  validate:
    name: 🔍 Lint & TypeCheck
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
      - run: npm ci
      - run: npm run lint
      - run: npx tsc --noEmit

  # ==================== 2. TESTING ====================
  test:
    name: 🧪 Unit & Integration Tests
    needs: validate
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
      - run: npm ci
      - run: npm run build
      - run: npm test -- --coverage
        env:
          NODE_ENV: test
          LOG_LEVEL: error
      - name: Upload Coverage Report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: coverage/
          retention-days: 14

  # ==================== 3. SECURITY SCAN ====================
  security-scan:
    name: 🛡️ Vulnerability Scan
    needs: validate
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: aquasecurity/trivy-action@master
        with:
          scan-type: 'fs'
          format: 'sarif'
          output: 'trivy-results.sarif'
          exit-code: '1'
          ignore-unfixed: true
          severity: 'CRITICAL,HIGH'
      - name: Upload Trivy scan results to GitHub Security
        if: always()
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: 'trivy-results.sarif'

  # ==================== 4. BUILD & PUSH IMAGES ====================
  build-push-images:
    name: 🐳 Build & Push Container Images
    needs: [test, security-scan]
    if: github.event_name == 'push'
    runs-on: ubuntu-latest
    strategy:
      matrix:
        agent: 
          - name: registry
            pkg: "@planner/registry-service"
          - name: planner
            pkg: "@planner/agent-planner"
          - name: builder
            pkg: "@planner/agent-builder"
          - name: qa
            pkg: "@planner/agent-qa"
          - name: deployer
            pkg: "@planner/agent-deployer"
          - name: monitoring
            pkg: "@planner/agent-monitoring"
          - name: incident
            pkg: "@planner/agent-incident-responder"
          - name: cost
            pkg: "@planner/agent-cost-optimizer"
          - name: compliance
            pkg: "@planner/agent-compliance-auditor"
          - name: notification
            pkg: "@planner/agent-notification"
      fail-fast: false
    permissions:
      contents: read
      packages: write
    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - uses: docker/build-push-action@v5
        with:
          context: .
          file: Dockerfile.agent
          push: true
          build-args: AGENT_PKG=${{ matrix.agent.pkg }}
          tags: |
            ${{ env.REGISTRY }}/agent-${{ matrix.agent.name }}:${{ github.sha }}
            ${{ env.REGISTRY }}/agent-${{ matrix.agent.name }}:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # ==================== 5. HELM PACKAGE & DEPLOY ====================
  package-deploy:
    name: 📦 Package & Deploy
    needs: build-push-images
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    environment: production
    steps:
      - uses: actions/checkout@v4
      - uses: azure/setup-helm@v3
        with:
          version: 'v3.13.0'
      - uses: azure/setup-kubectl@v3
        with:
          version: 'v1.28.2'

      - name: Configure kubeconfig
        run: |
          mkdir -p $HOME/.kube
          echo "${{ secrets.KUBE_CONFIG_BASE64 }}" | base64 -d > $HOME/.kube/config

      - name: Helm Package & Push to OCI
        run: |
          helm package ${{ env.HELM_CHART_PATH }} --version ${{ github.sha }} --destination ./dist/charts
          helm push ./dist/charts/planner-ecosystem-${{ github.sha }}.tgz oci://${{ env.REGISTRY }}/charts

      - name: Deploy to Production
        run: |
          helm upgrade --install planner-ecosystem \
            oci://${{ env.REGISTRY }}/charts/planner-ecosystem \
            --version ${{ github.sha }} \
            --namespace planner-system \
            --create-namespace \
            --wait --timeout=10m \
            --values ${{ env.HELM_CHART_PATH }}/values.yaml \
            --set global.image.tag=${{ github.sha }}
```

✅ Kaydedin: `.github/workflows/ci-cd.yml`

# 🔑 GEREKLİ GITHUB REPO AYARLARI

Pipeline'ın çalışabilmesi için repository ayarlarında şu adımları yapmanız gerekir:

1. **Environment Oluştur:**
   - `Settings > Environments > New environment` → `production`
   - `Protection rules` → `Required reviewers` (opsiyonel, onaylı deployment için)
   - `Environment secrets` bölümüne aşağıdaki secret'ları ekleyin:
     - `KUBE_CONFIG_BASE64`: `kubectl config view --raw | base64` komutunun çıktısı
     - `REGISTRY_PASSWORD`: (GitHub Token otomatik `$GITHUB_TOKEN` olarak kullanılır, ek gerekmez)

2. **Repository Permissions:**
   - `Settings > Actions > General > Workflow permissions` → `Read and write permissions`
   - `Allow GitHub Actions to create and approve pull requests` → ✅ İşaretli

### 📄 1. `monitoring/prometheus/rules.yml`
SLO bazlı, severity seviyeli, runbook linkli alarm kuralları.

```yaml
# monitoring/prometheus/rules.yml
groups:
  - name: planner_agent_slos
    interval: 15s
    rules:
      - alert: AgentDown
        expr: up{job=~"agent-.*"} == 0
        for: 2m
        labels:
          severity: critical
          team: platform
        annotations:
          summary: "Agent {{ $labels.instance }} is down"
          description: "Agent {{ $labels.job }} has been unreachable for over 2 minutes."
          runbook_url: "https://docs.planner.io/runbooks/RB-001-AgentDown"

      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5..", job=~"agent-.*"}[5m]) / rate(http_requests_total{job=~"agent-.*"}[5m]) > 0.05
        for: 3m
        labels:
          severity: warning
          team: sre
        annotations:
          summary: "High error rate on {{ $labels.job }}"
          description: "More than 5% of requests are failing with 5xx status codes."

      - alert: HighLatencyP95
        expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket{job=~"agent-.*"}[5m])) > 2
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High P95 latency on {{ $labels.job }}"
          description: "P95 response time is above 2 seconds."

      - alert: SagaFailureRate
        expr: rate(saga_executions_total{status="failed"}[10m]) / rate(saga_executions_total[10m]) > 0.1
        for: 5m
        labels:
          severity: critical
          team: workflow
        annotations:
          summary: "High Saga failure rate"
          description: "Over 10% of distributed workflows are failing in the last 10 minutes."

      - alert: AuditChainTampering
        expr: audit_chain_verification_failures_total > 0
        for: 0m
        labels:
          severity: critical
          team: security
        annotations:
          summary: "Audit chain integrity compromised"
          description: "Cryptographic verification of the audit log chain has failed. Immediate security review required."
```

# 🔗 DOCKER COMPOSE & K8S ENTEGRASYONU

### Docker Compose'a Ekleyin (`docker-compose.yml` içine)
```yaml
  prometheus:
    image: prom/prometheus:latest
    container_name: planner-prometheus
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.retention.time=14d'
    volumes:
      - ./monitoring/prometheus/rules.yml:/etc/prometheus/rules.yml:ro
      - prom_data:/prometheus
    ports: ["9090:9090"]
    networks: [planner-net]

  grafana:
    image: grafana/grafana:latest
    container_name: planner-grafana
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=${GRAFANA_PASSWORD:-admin}
      - GF_USERS_ALLOW_SIGN_UP=false
    volumes:
      - ./monitoring/grafana/provisioning:/etc/grafana/provisioning
      - ./monitoring/grafana/dashboards:/var/lib/grafana/dashboards
      - grafana_data:/var/lib/grafana
    ports: ["3000:3000"]
    networks: [planner-net]
    depends_on: [prometheus]

volumes:
  prom_data:
  grafana_data:
```

## 🛠️ Kullanılabilir Komutlar
| Komut | Açıklama |
|-------|----------|
| `make start` | Tüm servisleri detached modda başlatır |
| `make stop` | Tüm servisleri graceful shutdown ile durdurur |
| `make down` | Container ve network'leri temizler |
| `make logs` | Tüm logları takip eder (`Ctrl+C` ile çık) |
| `make logs-agent AGENT=planner` | Belirli bir ajanın logunu takip eder |
| `make health` | Registry üzerinden ajan sağlık durumunu sorgular |
| `make test` | Workspace unit testlerini çalıştırır |
| `make deploy` | Test → Build → Start → Health zincirini çalıştırır |
| `make clean` | Container, volume ve `dist/` klasörlerini temizler |
| `./scripts/verify-stack.sh` | Deployment sonrası otomatik doğrulama |

## ⚙️ Yapılandırma

### Geliştirme Ortamı
`.env` dosyasındaki değişkenler `docker-compose.yml` tarafından otomatik enjekte edilir.
- `LOG_LEVEL=debug` → Detaylı trace almak için
- `NATS_URL=nats://localhost:4222` → Yerel mesajlaşma
- `REGISTRY_URL=http://localhost:3001` → Yerel servis keşif

### Production Ortamı
`packages/infra/helm/planner-ecosystem/values.yaml` üzerinden yönetilir.
- Secret'lar için `External Secrets Operator` veya `Vault Agent Injector` kullanın
- `global.image.tag` → CI/CD tarafından otomatik güncellenir (`${GITHUB_SHA}`)
- `autoscaling.enabled` → CPU/Memory bazlı yatay ölçeklendirmeyi açar

## 🔍 Debug & Teşhis

### Ajan Çalışmıyorsa
```bash
docker compose logs -f agent-<isim>
# Örnek hata: EADDRINUSE → PORT çakışması, .env veya docker-compose.yml'de portu değiştirin
# Örnek hata: NATS connection refused → nats servisinin healthy olduğundan emin olun: docker compose ps nats
```

### Saga Başarısız Olursa
1. Grafana > "Saga Success vs Failed" panelini kontrol edin
2. `curl http://localhost:3025/api/v1/saga/<sagaId>` (IncidentResponder logs)
3. Audit zincirini doğrulayın: `planner debug audit --limit=50`

### Registry'e Kayıt Sorunu
```bash
curl -sf http://localhost:3001/api/v1/agents | jq '.agents[] | {agent_id, status}'
# Status "offline" görünürsa, ajanın heartbeat interval'ı ve network erişimini kontrol edin
```
